# API Security Risk Analysis

## Overview

This project analyzes security risks in publicly available API endpoints using Postman and security assessment techniques.

## API Tested

* JSONPlaceholder API
* Endpoint: /users
* Endpoint: /posts

## Tools Used

* Postman
* Browser Developer Tools

## Methodology

1. Reviewed API documentation.
2. Sent requests using Postman.
3. Examined response data and headers.
4. Identified security risks.
5. Classified risk severity.
6. Suggested remediation measures.

## Findings

* Unauthenticated Access (Medium)
* Excessive Data Exposure (Medium)
* Weak Access Control Observation (Medium)
* Rate Limiting Review (Low)
